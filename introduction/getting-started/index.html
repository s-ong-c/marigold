Getting Started

This page describes how to get started building an application with Marigold.

Installation

To install Marigold you can use any common package managers like yarn pnpm or npm.

With npm:

npm install @marigold/components --save

With yarn:

yarn add @marigold/components

With pnpm:

pnpm add @marigold/components

In addition to the components, you usually want to install an existing theme.

Note

Please note that react >= 17.0.0 and react-dom >= 17.0.0 are peer dependencies for @marigold/components.

Tooling

Marigold works with popular build tools, like webpack, rollup and esbuild.

If you are using TypeScript to build your app, be sure to add @types/react and @types/react-dom to your project. Marigold is written in TypeScript and Marigold components are published with their type definitions so that you can take advantage of static type checking and IDE autocomplete.

Bootstrapping

For Marigold to work correctly, you need to wrap your app with the MarigoldProvider. This way Marigold components will be able to access styles and SSR will work out of the box.

import React from 'react';
import { Button, MarigoldProvider } from '@marigold/components';
import theme from '@marigold/theme-unicorn';


const App = () => (
  <MarigoldProvider theme={theme}>
    <Button variant="primary" onClick={() => alert('Hey there!')}>
      Hello World
    </Button>
  </MarigoldProvider>
);

See the Provider and Button docs for more information about the components used in this example.
